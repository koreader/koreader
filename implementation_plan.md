# План поэтапной реализации кросс-форматной синхронизации закладок

Этот план описывает создание плагина `bookmarks_sync` для KOReader, который позволит накапливать, нормализовать и переносить закладки и выделения между различными форматами одной и той же книги (например, EPUB, FB2, MOBI, PDF).

Синхронизация построена на базе концепции **текстовых якорей (Text Quote Selector)** — привязке к тексту и контексту вместо жестких координат файлов.

---

## Этапы реализации

### Этап 1. Сбор данных и нормализация (Локальный экспорт)
Цель: Научить плагин перехватывать изменения закладок, формировать для них текстовые якоря и сохранять в файл `bookmarks_sync.lua` внутри SDR-папки книги.

1. **Нормализация текста и вычисление ID книги:**
   - Написание функции нормализации строк (приведение к нижнему регистру, замена `ё` на `е`, удаление диакритики, пунктуации и лишних пробелов).
   - Вычисление идентификатора книги (`book_id`) на основе базового имени файла (без пути и расширения): `md5(base_name)`. Это позволит сопоставлять файлы `book.epub` и `book.pdf`.
2. **Унифицированный захват контекста (Prefix / Exact / Suffix):**
   - Для всех форматов (EPUB, FB2, MOBI, PDF, DjVu) извлечение контекста будет производиться через вызов единого системного метода документа: `self.ui.document:getSelectedWordContext(exact, nb_words, pos0, pos1)`.
3. **Локальное хранилище в SDR:**
   - Определение директории сохранения с помощью `DocSettings:getSidecarDir(doc_path)` (уважает системные настройки KOReader).
   - Запись файла `bookmarks_sync.lua` внутри SDR-папки книги. При открытии книги плагин проверяет, совпадает ли имя файла с сохраненным `current_basename`, и при отличии записывает старое имя в массив `basenames_history` (история переименований).

### Этап 2. Импорт и применение закладок (Восстановление в другом формате)
Цель: Научить плагин при открытии книги искать сохраненные закладки для этого произведения, сопоставлять их с текстом текущего формата и воссоздавать их.

1. **Поиск совпадений книг с учетом переименования:**
   - При открытии книги (`onReaderReady`), плагин вычисляет `book_id` текущего файла.
   - Плагин ищет файлы `bookmarks_sync.lua` во всех SDR-директориях в истории книг (или в центральной папке настроек).
   - Совпадение засчитывается, если текущее базовое имя файла совпадает с `current_basename` или содержится в списке `basenames_history` внутри чужого файла `bookmarks_sync.lua`.
2. **Локальный поиск текста якоря:**
   - Для каждой найденной внешней закладки плагин берет значение `progress` (например, 0.45), переходит в эту область документа и запускает поиск строки `exact` через `self.ui.document:findText()`.
   - Если найдено совпадение, проверяется соответствие контекста `prefix` и `suffix`.
3. **Воссоздание закладки:**
   - При успешном нахождении текста в новом формате плагин программно создает локальную закладку/выделение (через вызов `self.ui.bookmark:toggleBookmark()` или `self.ui.highlight:addHighlight()`).

### Этап 3. Мягкое удаление (Soft Delete) и синхронизация изменений
Цель: Обеспечить корректную работу при удалении закладок и подготовке к облачной синхронизации.

1. **Реализация Soft Delete:**
   - При удалении закладки в KOReader плагин не удаляет её из файла `bookmarks_sync.lua` сразу, а выставляет флаг `deleted = true` и временную метку `datetime_deleted`.
2. **Откат удалений (Undo):**
   - Реализация временного буфера в памяти для восстановления случайно удаленной закладки кнопкой "Отменить".

### Этап 4. Сетевая синхронизация (Kosync / WebDAV)
Цель: Отправка и получение файла базы данных закладок с удаленного сервера в неблокирующем режиме.

---

## Архитектура плагина

### Основные компоненты плагина:
1. `_meta.lua` — Регистрация в KOReader.
2. `main.lua` — Event-listener событий (`onAnnotationsModified`, `onSaveSettings`, `onReaderReady`), интеграция с меню настроек.
3. `anchoring.lua` [NEW] — Модуль для извлечения контекста, нормализации текстов и поиска совпадений.
4. `sync_db.lua` [NEW] — Модуль для работы с базой данных закладок на диске.

## План верификации

### Ручное тестирование

#### Тест 1: Сбор контекста (EPUB и PDF)
1. Открыть книгу формата EPUB, выделить предложение, проверить запись в `bookmarks_sync.lua` (наличие `exact`, `prefix` и `suffix`).
2. Открыть ту же книгу в формате PDF, сделать то же выделение, убедиться в корректности структуры `bookmarks_sync.lua`.

#### Тест 2: Проверка истории переименований
1. Переименовать книгу `book.epub` в `book_new.epub`.
2. Открыть её в KOReader, убедиться, что старое имя `book` переместилось в `basenames_history` внутри `bookmarks_sync.lua`.

#### Тест 3: Кросс-форматный перенос
1. Создать закладку в EPUB версии книги.
2. Открыть ту же книгу в формате PDF (с именем `book.pdf` или `book_new.pdf`).
3. Убедиться, что плагин обнаружил закладку из EPUB по имени/истории имен, нашел нужный текст по контексту и автоматически воссоздал её в PDF.
